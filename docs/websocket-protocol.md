# WebSocket Protocol

Voicer WebSocket-сервер слушает на `ws://0.0.0.0:{port}` (по умолчанию порт 5050, настраивается). К серверу может подключаться любое количество клиентов одновременно.

## Claim-протокол

Только один клиент одновременно является **активным** получателем транскрипций. Это предотвращает дублирование — например, когда несколько экземпляров OpenCode подключены к одному серверу.

- Первый подключённый клиент автоматически становится активным
- Любой клиент может отправить `claim`, чтобы стать активным (последний claim выигрывает)
- При отключении активного клиента роль автоматически передаётся следующему подключённому
- При `release` роль также передаётся следующему клиенту (если есть)
- Транскрипции отправляются **только** активному клиенту
- Статусы и ошибки транслируются **всем** клиентам

## Сообщения: Клиент -> Сервер

### claim

Запрос на получение роли активного клиента. Предыдущий активный клиент получит уведомление `claimed` с `active: false`.

```json
{"type": "claim"}
```

### release

Добровольный отказ от роли активного клиента. Роль автоматически передаётся следующему подключённому клиенту (если есть).

```json
{"type": "release"}
```

## Сообщения: Сервер -> Клиент

### claimed

Подтверждение статуса claim. Отправляется при подключении, после `claim`/`release`, и при смене активного клиента.

```json
{"type": "claimed", "active": true}
{"type": "claimed", "active": false}
```

### transcription

Распознанный текст. **Отправляется только активному клиенту.**

```json
{"type": "transcription", "text": "Распознанный текст.", "timestamp": "2025-01-15T12:30:00.000Z"}
```

Если перед нажатием горячей клавиши (F6) был выделен текст и включена опция «Include selected text in WS mode», выделенный текст добавляется в начало поля `text` через пробел:

```json
{"type": "transcription", "text": "выделенный текст Надиктованный текст.", "timestamp": "2025-01-15T12:30:00.000Z"}
```

### status

Изменение состояния микрофона. **Отправляется всем клиентам.**

```json
{"type": "status", "status": "recording"}
{"type": "status", "status": "processing"}
{"type": "status", "status": "idle"}
```

| status | Значение |
|--------|----------|
| `idle` | Ожидание, микрофон выключен |
| `recording` | Запись — пользователь удерживает горячую клавишу |
| `processing` | Распознавание речи в процессе |

### error

Ошибка распознавания или другая ошибка. **Отправляется всем клиентам.**

```json
{"type": "error", "message": "Описание ошибки"}
```

## Пример взаимодействия

```
Клиент A подключается       -> Сервер отправляет A: {"type":"claimed","active":true}
Клиент B подключается       -> Сервер отправляет B: {"type":"claimed","active":false}
Клиент B отправляет claim   -> Сервер отправляет A: {"type":"claimed","active":false}
                             -> Сервер отправляет B: {"type":"claimed","active":true}
Пользователь диктует        -> Сервер отправляет B: {"type":"transcription","text":"..."}
                             -> Сервер отправляет A и B: {"type":"status","status":"..."}
Клиент B отключается        -> Сервер автоматически передаёт роль A
                             -> Сервер отправляет A: {"type":"claimed","active":true}
Клиент A отправляет release  -> Сервер отправляет A: {"type":"claimed","active":false}
                             -> Нет других клиентов — activeClient = null
```

## Подключение из WSL2

Из WSL2 Windows-хост доступен по IP из `/etc/resolv.conf`:

```bash
export VOICER_URL="ws://$(grep nameserver /etc/resolv.conf | awk '{print $2}'):5050"
```
